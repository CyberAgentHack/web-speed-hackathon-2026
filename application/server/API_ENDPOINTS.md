# API エンドポイント一覧

ベースパス: `/api/v1`

フレームワーク: Express + express-session (MemoryStore) + Sequelize (SQLite)

認証方式: セッションベース認証 (`req.session.userId`)

---

## 1. 認証 (auth)

### POST `/api/v1/signup`

| 項目             | 内容                                                                                                                                                                                                 |
| ---------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 認証             | 不要                                                                                                                                                                                                 |
| 責務             | 新規ユーザー登録し、セッションにログイン状態を保存する                                                                                                                                               |
| リクエストボディ | `{ username: string, name: string, password: string }`                                                                                                                                               |
| レスポンス (200) | User オブジェクト (profileImage 含む、password は除外)                                                                                                                                               |
| 4xx エラー       | **400** `{ code: "USERNAME_TAKEN" }` — username が既に使われている (UniqueConstraintError)<br>**400** `{ code: "INVALID_USERNAME" }` — username が `/^[a-z0-9_-]+$/i` に合致しない (ValidationError) |

### POST `/api/v1/signin`

| 項目             | 内容                                                           |
| ---------------- | -------------------------------------------------------------- |
| 認証             | 不要                                                           |
| 責務             | ログイン。セッションにユーザーIDを保存する                     |
| リクエストボディ | `{ username: string, password: string }`                       |
| レスポンス (200) | User オブジェクト                                              |
| 4xx エラー       | **400** — ユーザーが見つからない<br>**400** — パスワードが不正 |

### POST `/api/v1/signout`

| 項目             | 内容                                         |
| ---------------- | -------------------------------------------- |
| 認証             | 不要 (セッションがあれば破棄する)            |
| 責務             | ログアウト。セッションから userId を削除する |
| リクエストボディ | なし                                         |
| レスポンス (200) | `{}`                                         |
| 4xx エラー       | なし                                         |

## 4. 検索 (search)

### GET `/api/v1/search`

| 項目               | 内容                                                                                                                                             |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------ |
| 認証               | 不要                                                                                                                                             |
| 責務               | 投稿をテキスト・ユーザー名・日付で検索する                                                                                                       |
| クエリパラメーター | `q: string` — 検索クエリ (特殊構文: `since:YYYY-MM-DD`, `until:YYYY-MM-DD`)<br>`limit?: number` — 取得件数上限<br>`offset?: number` — オフセット |
| レスポンス (200)   | Post オブジェクトの配列 (createdAt 降順、テキスト一致とユーザー名/名前一致をマージ・重複排除)                                                    |
| 4xx エラー         | なし (クエリが空の場合は空配列 `[]` を返す)                                                                                                      |

**検索クエリ構文:**

- 通常のテキスト → 投稿テキスト・ユーザー名・ユーザー表示名を LIKE 検索
- `since:2024-01-01` → 指定日以降
- `until:2024-12-31` → 指定日以前
- これらは組み合わせ可能

## 5. ダイレクトメッセージ (direct_message)

### GET `/api/v1/dm`

| 項目             | 内容                                                                                   |
| ---------------- | -------------------------------------------------------------------------------------- |
| 認証             | **必要**                                                                               |
| 責務             | ログインユーザーが参加している DM 会話一覧を取得する (メッセージが1件以上ある会話のみ) |
| レスポンス (200) | DirectMessageConversation の配列 (initiator, member, messages 含む。最新メッセージ順)  |
| 4xx エラー       | **401** — 未ログイン                                                                   |

### POST `/api/v1/dm`

| 項目             | 内容                                                             |
| ---------------- | ---------------------------------------------------------------- |
| 認証             | **必要**                                                         |
| 責務             | 指定ユーザーとの DM 会話を取得、または存在しなければ新規作成する |
| リクエストボディ | `{ peerId: string }` — 相手ユーザーの ID                         |
| レスポンス (200) | DirectMessageConversation オブジェクト                           |
| 4xx エラー       | **401** — 未ログイン<br>**404** — peerId のユーザーが存在しない  |

### GET `/api/v1/dm/:conversationId`

| 項目             | 内容                                                                         |
| ---------------- | ---------------------------------------------------------------------------- |
| 認証             | **必要**                                                                     |
| 責務             | 指定 ID の DM 会話を取得する (自分が参加者であること)                        |
| パスパラメーター | `conversationId` — 会話 ID (UUID)                                            |
| レスポンス (200) | DirectMessageConversation オブジェクト (initiator, member, messages 含む)    |
| 4xx エラー       | **401** — 未ログイン<br>**404** — 会話が存在しない、または自分が参加者でない |

### POST `/api/v1/dm/:conversationId/messages`

| 項目             | 内容                                                                                                                      |
| ---------------- | ------------------------------------------------------------------------------------------------------------------------- |
| 認証             | **必要**                                                                                                                  |
| 責務             | DM 会話にメッセージを送信する                                                                                             |
| パスパラメーター | `conversationId` — 会話 ID                                                                                                |
| リクエストボディ | `{ body: string }` — メッセージ本文                                                                                       |
| レスポンス (201) | 作成された DirectMessage オブジェクト (sender 含む)                                                                       |
| 4xx エラー       | **401** — 未ログイン<br>**400** — body が空文字または文字列でない<br>**404** — 会話が存在しない、または自分が参加者でない |

### POST `/api/v1/dm/:conversationId/read`

| 項目             | 内容                                                                         |
| ---------------- | ---------------------------------------------------------------------------- |
| 認証             | **必要**                                                                     |
| 責務             | DM 会話の未読メッセージを既読にする (相手からのメッセージのみ)               |
| パスパラメーター | `conversationId` — 会話 ID                                                   |
| レスポンス (200) | `{}`                                                                         |
| 4xx エラー       | **401** — 未ログイン<br>**404** — 会話が存在しない、または自分が参加者でない |

### POST `/api/v1/dm/:conversationId/typing`

| 項目             | 内容                                               |
| ---------------- | -------------------------------------------------- |
| 認証             | **必要**                                           |
| 責務             | タイピング中通知を相手に送信する (EventHub 経由)   |
| パスパラメーター | `conversationId` — 会話 ID                         |
| レスポンス (200) | `{}`                                               |
| 4xx エラー       | **401** — 未ログイン<br>**404** — 会話が存在しない |

## 8. Crok (AI Q&A)

### GET `/api/v1/crok/suggestions`

| 項目             | 内容                         |
| ---------------- | ---------------------------- |
| 認証             | 不要                         |
| 責務             | Q&A の質問候補一覧を取得する |
| レスポンス (200) | `{ suggestions: string[] }`  |
| 4xx エラー       | なし                         |

### GET `/api/v1/crok`

| 項目               | 内容                                                                                           |
| ------------------ | ---------------------------------------------------------------------------------------------- |
| 認証               | **必要**                                                                                       |
| 責務               | AI 回答を SSE (Server-Sent Events) でストリーミング返却する                                    |
| レスポンスヘッダー | `Content-Type: text/event-stream`                                                              |
| SSE イベント       | `event: message`, `data: { text: string, done: boolean }` — 1文字ずつ送信。最後に `done: true` |
| 備考               | TTFT (Time to First Token) として 3秒の遅延あり。1文字あたり 10ms の遅延                       |
| 4xx エラー         | **401** — 未ログイン                                                                           |

## 共通事項

### エラーハンドリング

グローバルエラーハンドラー (`routes/api.ts`) により以下の処理が行われる:

1. **Sequelize ValidationError** → `400 Bad Request` に変換
2. **http-errors** によるエラー → そのステータスコードで返却
3. その他のエラー → `500 Internal Server Error`

全エラーレスポンスの形式: `{ message: string }`

### レスポンスヘッダー

全レスポンスに共通で付与:

- `Cache-Control: max-age=0, no-transform`
- `Connection: close`

### 静的ファイル配信

`/api/v1` 以外のパスは静的ファイル配信 + SPA フォールバック (`connect-history-api-fallback`):

1. アップロードディレクトリ (`UPLOAD_PATH`)
2. パブリックディレクトリ (`PUBLIC_PATH`)
3. クライアントビルド成果物 (`CLIENT_DIST_PATH`)
