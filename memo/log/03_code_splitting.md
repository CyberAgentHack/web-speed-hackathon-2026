# ルートベースのコード分割 (React.lazy)

## 変更内容

- `AppContainer.tsx`: 9つのルートコンテナを `React.lazy()` + `dynamic import()` に変更
- `<Routes>` を `<Suspense fallback={null}>` で囲む
- 初期バンドル: **12MB → 722KB** に削減
- ref: https://react.dev/reference/react/lazy

### 分割後のチャンク構成

| チャンク                  | サイズ |
| ------------------------- | ------ |
| index (初期)              | 722KB  |

| SearchContainer           | 4.3MB  |
| CrokContainer             | 1.3MB  |
| ffmpeg-core               | 110KB  |
| lodash                    | 70KB   |
| kuromoji                  | 67KB   |
| moment                    | 59KB   |
| TermContainer             | 18KB   |
| その他                    | 各数KB |

## 分析

- DM・利用規約で TBT が **30点満点** に到達（初期バンドルが小さくルート固有の依存が軽い）
- SI も 0.10〜0.20 点入り始めた
- ホーム・投稿詳細系は依然 FCP/LCP/SI = 0 → `use_infinite_fetch` (5.4MB) が共有チャンクとして読み込まれるため
- 次の改善: use_infinite_fetch チャンクの分解、重いライブラリ (moment, lodash, kuromoji) の置換・遅延読み込み
