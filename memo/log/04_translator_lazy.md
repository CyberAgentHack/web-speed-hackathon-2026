# web-llm (翻訳) の dynamic import 化

## 変更内容

- `TranslatableText.tsx`: `createTranslator` の静的 import を、クリック時の dynamic import に変更
- `@mlc-ai/web-llm` (13MB) が初期バンドルから分離され、「Show Translation」クリック時のみ読み込まれるように
- 共有チャンク `use_infinite_fetch` が **5.4MB → 109KB** に激減

### チャンク構成の変化

| チャンク | 変更前 | 変更後 |
|---|---|---|
| use_infinite_fetch (共有) | 5.4MB | 109KB |
| create_translator (遅延) | - | 5.3MB |
| index (初期) | 722KB | 723KB |

## 計測結果

合計 **339.70 / 1150 点** （前回 334.75 → **+4.95**）

## 分析

- Crok AIチャットの INP が 25点満点に（web-llm が初期読み込みから外れた効果）
- 通常テストの SI に 0.10 点が入り始めた（複数ページ）
- FCP/LCP は依然 0 点 → サーバー側の応答速度 or gzip 圧縮が次の改善ポイント
- ユーザーフロー失敗（認証・検索・投稿）は flaky で、コード変更起因ではない
